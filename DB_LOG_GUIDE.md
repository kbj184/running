# 러닝 앱 DB 로그 확인 가이드

## 📊 DB 저장 로그 확인 방법

러닝을 시작하면 브라우저 콘솔에서 IndexedDB에 저장되는 모든 데이터를 실시간으로 확인할 수 있습니다.

### 1️⃣ 브라우저 개발자 도구 열기

**Windows/Linux:**
- `F12` 또는 `Ctrl + Shift + I`

**Mac:**
- `Cmd + Option + I`

### 2️⃣ Console 탭 선택

개발자 도구에서 **Console** 탭을 클릭합니다.

### 3️⃣ 러닝 시작

앱에서 **시작** 버튼을 클릭하면 다음과 같은 로그가 표시됩니다:

## 🔍 저장되는 데이터 항목

각 저장 시점마다 다음 정보가 IndexedDB에 저장됩니다:

| 항목 | 설명 | 예시 |
|------|------|------|
| **sessionId** | 세션 고유 ID | `session_1733281417123` |
| **timestamp** | 저장 시간 (밀리초) | `1733281417123` |
| **position** | GPS 좌표 [위도, 경도] | `[37.566500, 126.978000]` |
| **distance** | 누적 거리 (km) | `2.450` |
| **speed** | 현재 속도 (km/h) | `7.35` |
| **pace** | 페이스 (분/km) | `8.16` |
| **duration** | 경과 시간 (초) | `1205` |
| **route** | 전체 경로 배열 | `[[37.566, 126.978], ...]` |
| **isComplete** | 완료 여부 | `true` / `false` |

## 💡 팁

1. **콘솔 필터링**: 콘솔에서 `💾` 또는 `IndexedDB`로 검색하면 저장 관련 로그만 볼 수 있습니다.

2. **저장 주기**: 기본적으로 2초마다 자동 저장됩니다.

3. **GPS 대기**: GPS 위치를 받기 전에는 `⏳ GPS 위치 대기 중... (저장 건너뜀)` 메시지가 표시됩니다.

4. **에러 확인**: 저장 실패 시 `❌` 아이콘과 함께 에러 메시지가 표시됩니다.

## 🗄️ IndexedDB 직접 확인

브라우저 개발자 도구에서:
1. **Application** 탭 (또는 **Storage** 탭) 선택
2. 왼쪽 메뉴에서 **IndexedDB** 확장
3. **RunningTrackerDB** → **runningSessions** 선택
4. 저장된 모든 레코드 확인 가능

## 🧪 테스트 시나리오

1. **시작 버튼 클릭** → 시작 로그 확인
2. **2초 대기** → 첫 번째 자동 저장 로그 확인
3. **4초 대기** → 두 번째 자동 저장 로그 확인
4. **종료 버튼 클릭** → 최종 저장 및 요약 로그 확인
