# 러닝 앱 테스트 모드 가이드

## 🧪 테스트 모드란?

실제 GPS 없이도 러닝 앱을 테스트할 수 있는 기능입니다. 마우스 클릭만으로 위치를 이동하며 DB 저장을 테스트할 수 있습니다.

## 🚀 사용 방법

### 1️⃣ 러닝 시작

1. 메인 화면에서 **시작** 버튼 클릭
2. 러닝 화면이 열리면 **기본적으로 테스트 모드**로 시작됩니다
3. 서울 중심 좌표에 초기 위치가 설정됩니다

### 2️⃣ 위치 이동 (테스트 모드)

1. **맵을 클릭**하면 해당 위치로 이동합니다
2. 클릭할 때마다:
   - 경로에 포인트가 추가됩니다
   - 거리가 자동 계산됩니다
   - 속도와 페이스가 업데이트됩니다
   - 콘솔에 로그가 출력됩니다

### 3️⃣ 모드 전환

화면 우측 상단의 버튼으로 모드를 전환할 수 있습니다:

- **🖱️ 테스트 모드**: 마우스 클릭으로 위치 이동
- **📍 GPS 모드**: 실제 GPS로 위치 추적

### 4️⃣ 데이터 저장 확인

1. **F12**를 눌러 개발자 도구 열기
2. **Console** 탭 선택
3. 2초마다 자동 저장되는 로그 확인

## 🎯 테스트 시나리오

### 시나리오 1: 기본 테스트
1. 시작 버튼 클릭
2. 맵을 3-4번 클릭하여 경로 생성
3. 콘솔에서 저장 로그 확인
4. 종료 버튼 클릭
5. 최종 요약 확인

### 시나리오 2: 긴 경로 테스트
1. 시작 버튼 클릭
2. 맵을 10번 이상 클릭하여 긴 경로 생성
3. 거리, 속도, 페이스 변화 관찰
4. 여러 번 저장되는 것 확인
5. 종료 후 결과 화면 확인

### 시나리오 3: 모드 전환 테스트
1. 테스트 모드로 시작
2. 몇 번 클릭하여 경로 생성
3. GPS 모드로 전환
4. 실제 GPS 데이터 수신 확인
5. 다시 테스트 모드로 전환

## 💡 팁

### 빠른 테스트
- 맵을 빠르게 여러 번 클릭하면 속도가 빨라집니다
- 멀리 떨어진 곳을 클릭하면 거리가 크게 증가합니다

### 정확한 테스트
- 천천히 클릭하여 실제 러닝과 유사한 속도 유지
- 경로를 따라 순차적으로 클릭

### 저장 확인
- 2초마다 자동 저장되므로 최소 2초 이상 대기
- 콘솔에서 `💾` 아이콘으로 저장 로그 필터링

## 🔍 IndexedDB 직접 확인

1. 개발자 도구 → **Application** 탭
2. **IndexedDB** → **RunningTrackerDB** → **runningSessions**
3. 저장된 모든 레코드 확인 가능

## ⚠️ 주의사항

- 테스트 모드에서는 GPS 권한이 필요 없습니다
- GPS 모드로 전환하면 위치 권한을 요청합니다
- 저장된 데이터는 브라우저의 IndexedDB에 영구 저장됩니다
- 브라우저 데이터를 삭제하면 저장된 기록도 삭제됩니다

## 🎨 UI 요소

### 모드 전환 버튼
- **위치**: 화면 우측 상단
- **테스트 모드**: 보라색 배경 (🖱️ 테스트 모드)
- **GPS 모드**: 초록색 배경 (📍 GPS 모드)

### 안내 메시지
- **위치**: 화면 하단 중앙
- **표시**: 테스트 모드일 때만
- **내용**: "🖱️ 맵을 클릭하여 위치를 이동하세요!"

### 통계 패널
- **거리**: 실시간 업데이트
- **시간**: 1초마다 증가
- **속도**: 거리/시간으로 자동 계산
- **페이스**: 시간/거리로 자동 계산
